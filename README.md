# NTP Extender

Небольшое приложение для внутренней сети предприятия:
- синхронизируется с несколькими внешними NTP-серверами в интернете;
- рассчитывает корректированное время (offset) без изменения системного времени контейнера;
- раздаёт это время клиентам во внутренней сети как локальный NTP-сервер (UDP `123`);
- предоставляет удобный веб-интерфейс для настройки, проверки источников и мониторинга.

Проект создан при поддержке AI-ассистента в программе Cursor.

## Что настраивается в интерфейсе

- внешние NTP-серверы (DNS или IP, до 10 источников);
- часовой пояс (IANA);
- интервал синхронизации;
- таймаут NTP-запроса;
- порт локального NTP-сервера;
- проверка доступности источников;
- просмотр статуса и логов.

## Авторизация

- логин/пароль по умолчанию после первого запуска: `admin / admin`;
- после первого входа рекомендуется сразу сменить пароль в интерфейсе.

## Развёртывание с Git на сервере (пошагово)

1. Подключитесь к серверу по SSH.
2. Убедитесь, что установлены Docker и Docker Compose plugin:

```bash
docker --version
docker compose version
```

3. Клонируйте репозиторий:

```bash
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ> ntp-extender
cd ntp-extender
```

4. Проверьте `docker-compose.yml` (можно запускать как есть).
5. Запустите сборку и контейнер:

```bash
docker compose up -d --build
```

6. Проверьте, что контейнер запущен:

```bash
docker compose ps
```

7. Откройте веб-интерфейс:

- `http://<IP_сервера>:12312`

8. Войдите с `admin / admin`, затем смените пароль.

## Готовый docker-compose.yml

Скопируйте этот `docker-compose.yml` и запустите приложение из Docker Hub.

```yaml
services:
  ntp-extender:
    image: arahdunakhor/ntp-extender:latest
    container_name: ntp-extender
    restart: unless-stopped
    ports:
      - "12312:12312"
      - "123:123/udp"
    volumes:
      - ./data:/data
    environment:
      - WEB_HOST=0.0.0.0
      - WEB_PORT=12312
      - NTP_PORT=123
```

Как запустить:

1. Создайте папку, например `ntp-extender`.
2. Вставьте в файл `docker-compose.yml` конфигурацию выше.
3. Выполните:

```bash
docker compose pull
docker compose up -d
```

## Обновление приложения

```bash
cd ntp-extender
git pull
docker compose up -d --build
```

## Логи и хранение

Логи хранятся в `./data/app.log` (на volume, не в слое контейнера).

Политика хранения логов:
- срок хранения: до `7` дней;
- ограничение суммарного объёма: до `50 MB`;
- старые файлы удаляются автоматически.
